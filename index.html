Comment utiliser le bisect ?

  1. On démarre la recherche : git bisect start
  2. On identifie le premier commit problématique connu : git bisect bad SHA-1 (par défaut HEAD)
  3. On identifie un commit où il n'y avait pas le problème : git bisect good SHA-1
  4. Git propose de tester des commits. On tape git bisect bad ou git bisect good
  5. Git trouve le commit qui a introduit le bug
